# 端口探查性能优化

## Goal
解决端口详情面板加载极慢的问题。实现缓存优先 + 增量查询策略。

## Requirements
- 点击端口详情时，首先从扫描器缓存读取基本信息立即渲染
- 后台仅查询该端口的增量信息（关联进程详情、连接状态），非全量扫描
- 查询期间显示骨架屏（skeleton），非空白 + 旋转图标
- 超时 3 秒自动降级为缓存数据 + "部分信息可能不是最新" 提示
- 切换端口时自动取消前一个查询（AbortController）
- 并行查询限制最大并行数 3

## Acceptance Criteria
- [ ] 点击端口详情 < 500ms 显示基本信息（缓存）
- [ ] 增量查询 < 3s 显示完整信息
- [ ] 超时后自动降级 + 提示
- [ ] 切换端口时前一个查询被取消
- [ ] 无 "Loading port data..." 长时间空白
- [ ] 骨架屏正确展示
