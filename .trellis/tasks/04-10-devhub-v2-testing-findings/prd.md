## 一、测试总结

DevHub 基础框架运行正常（Electron 启动、React 渲染、IPC 通信），但在 **进程管理深度**、**端口拓扑完整性**、**窗口管理核心功能**、**主题视觉体系** 四个维度存在系统性不足。以下按优先级排列。

---

## 二、问题清单与改进需求

### P0 — 窗口管理（重中之重）

#### 2.1 AI 窗口自命名与通知标识

**现状问题：** 用户同时开启多个 AI CLI 工具实例（Claude Code × N、Codex CLI、OpenCode、Gemini CLI 等），任务完成通知无法区分来源。

**需求：**
- [ ] AI 窗口支持 **用户自定义命名**（如 "Claude-前端重构"、"Codex-API开发"）
- [ ] 任务完成通知 **必须携带窗口名称**，格式示例：`[Claude-前端重构] 任务完成`
- [ ] 窗口列表中 AI 窗口 **优先显示 / 特殊标记**，与普通窗口区分
- [ ] 支持通过名称快速定位和聚焦对应窗口

#### 2.2 分组与布局功能

**现状问题：** "分组"和"布局"功能不可用，行为异常。

**需求：**
- [ ] 调查并修复分组功能（按类型/自定义分组）
- [ ] 调查并修复布局功能（排列/平铺/层叠）
- [ ] 确保功能按钮状态与实际行为一致

#### 2.3 监控进度

**现状问题：** 进度监控不够精确和实用。

**需求：**
- [ ] 增强 AI 任务进度检测精度
- [ ] 提供更丰富的进度状态（空闲/思考中/编码中/等待输入/完成）
- [ ] 进度变化的历史时间线视图

### P0 — Bug 修复

#### 2.4 focusWindow PowerShell 兼容性

**现状问题：** `focusWindow` 调用 PowerShell `Add-Type` 编译 C# 代码时失败，因为使用了 C# 7+ 语法 `out _`（discard），而 Windows PowerShell 5.1 自带的编译器仅支持 C# 5。

**需求：**
- [ ] 将 `out _` 改为 `out uint dummy` 等 C# 5 兼容写法
- [ ] 验证 PowerShell 输出编码与 Node.js child_process 的兼容性
- [ ] 添加 PowerShell 版本检测或回退机制

---

### P1 — 进程管理

#### 2.5 进程关系图：从静态拓扑到动态神经流线图

**现状问题：** 进程关联图仅为静态拓扑，缺乏动态性和生命力。

**需求：**
- [ ] 进程关系图改为 **动态关系流线图**，类似神经网络图
  - 节点间连线有动态流动动画（数据/信号流向可视化）
  - 节点大小/颜色随 CPU/内存占用动态变化
  - 连线粗细反映进程间通信频率或依赖强度
- [ ] 支持实时更新，进程创建/销毁时图自动演化
- [ ] 支持缩放、拖拽、聚焦交互

#### 2.6 进程列表显示优化

**现状问题：** 进程排列显示差于 Windows 资源管理器。

**需求：**
- [ ] 支持 **升序/降序排序**（按 PID、CPU%、内存、名称等字段）
- [ ] 支持 **真实进程勘探**（读取进程的详细信息：命令行参数、工作目录、环境变量、启动时间、DLL 依赖等）
- [ ] **进程树重新定义**：不仅展示父子关系，而是梳理与该进程有关的 **所有关联进程 + 端口 + 网络连接**，构建完整的动态关系神经图
- [ ] **进程卡片增强**：
  - 提高信息密度（CPU/内存/线程数/句柄数/IO 读写/网络连接数）
  - 增加操作按钮（结束进程/结束进程树/设置优先级/打开文件位置）
  - 展开详情面板（命令行、环境变量、模块列表、网络连接）

---

### P1 — 端口拓扑

#### 2.7 端口拓扑完善

**现状问题：** 端口拓扑独立展示很好，但功能不够完善。

**需求：**
- [ ] 支持 **聚焦到单个端口**，展开该端口的完整关联视图
- [ ] 端口关联维度：
  - 占用该端口的进程（及其进程树）
  - 监听/已连接状态
  - 远程地址与连接状态
  - 网络流量统计（如有权限）
- [ ] 端口与进程的交叉关系图（同样采用神经流线图风格）
- [ ] 端口冲突检测高亮

---

### P2 — 主题系统

#### 2.8 主题不是换颜色

**现状问题：** 主题切换仅改变颜色，缺乏视觉传达的本质差异。

**需求：**
- [ ] 主题应包含 **视觉传达体系** 的全面差异：
  - 组件排布/布局策略（紧凑型 vs 宽松型、卡片 vs 列表）
  - 字体家族与排版节奏（衬线 vs 无衬线、行高、字间距）
  - 圆角策略（方正 vs 圆润）
  - 阴影与层次表现（扁平 vs 拟物 vs 玻璃态）
  - 动效风格（苏维埃刚健 vs 赛博流畅 vs 极简克制）
  - 信息密度与留白比例
- [ ] 每个主题是一套 **完整的设计语言**，而非一张调色板
- [ ] 至少提供 2-3 套有本质差异的主题：
  - Soviet Constructivism（当前主题，增强几何感与力量感）
  - Cyberpunk / Neon（赛博朋克，发光边缘，全息元素）
  - Minimal / Swiss（极简瑞士风，极致留白与网格）
