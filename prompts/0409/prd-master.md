# DevHub V2 - 系统级增强 PRD

> **版本**: 2.0 | **日期**: 2026-04-09 | **作者**: ZRainbow
> **状态**: Draft | **优先级**: P0

---

## 一、背景与动机

DevHub 当前版本已实现基础的进程/端口/窗口监控与 AI 任务追踪功能。但在实际使用中暴露出以下核心问题：

1. **信息密度不足** — 监控面板数值显示无法随窗口缩放自适应，浪费空间或溢出截断
2. **关系洞察缺失** — 进程/端口拓扑仅为静态层级图，无法呈现动态、流式的神经网络般关联
3. **操控性不足** — 进程列表缺少排序/过滤/真实勘探能力，卡片信息密度低于 Windows 任务管理器
4. **端口拓扑孤立** — 无法聚焦到单个端口进行深度关联分析（关联进程、网络流量等）
5. **窗口管理核心缺陷** — AI 窗口无法自命名、任务完成通知缺少窗口标识、分组/布局功能不可用
6. **主题仅为换色** — 缺乏真正的视觉传达差异化（组件排布、信息层级、装饰元素应随主题变化）

---

## 二、目标

### 2.1 核心目标

| # | 目标 | 衡量标准 |
|---|------|----------|
| G1 | 监控面板在任意窗口尺寸下信息完整且美观 | 400px~2560px 宽度范围内无溢出/截断 |
| G2 | 进程/端口关系以动态神经流线图呈现 | 支持实时动画、力导引布局、脉冲流线 |
| G3 | 进程管理能力不低于 Windows 资源管理器 | 支持升降序、多列排序、搜索过滤、进程树全关联 |
| G4 | 端口拓扑支持单端口聚焦与深度关联 | 点击端口可展开关联进程/连接/流量面板 |
| G5 | AI 窗口可自命名，通知携带窗口名 | 用户可为每个 AI CLI 实例设定别名 |
| G6 | 主题差异化体现在布局/排版/装饰/动效 | 3 个主题有明显视觉传达差异 |

### 2.2 非目标

- 跨平台支持（本次仅 Windows）
- 新增功能模块（聚焦现有模块增强）
- 性能基准测试（但不能引入明显卡顿）

## 四、各领域概要

### 4.1 Spec-01: 监控面板响应式增强

**问题**: `HeroStats` 和 `StatCard` 使用 `clamp(14px, 2.5vw, 24px)` 做响应，但在极窄/极宽窗口下数值截断或间距不当。hero-number 在 globals.css 中有三级断点但过渡生硬。

**方案方向**:
- 引入 CSS Container Queries 替代 viewport-based clamp
- 动态字号 + 动态间距 + 自动省略策略
- 数值格式化随空间变化（完整数字 → 缩写 → 图标+数字）
- 统一的 `ResponsiveMetric` 组件

**涉及文件**: `HeroStats.tsx`, `StatCard.tsx`, `globals.css`, `MonitorPanel.tsx`

---

### 4.2 Spec-02: 进程管理系统级增强

**问题**: ProcessView 1043 行但无排序/搜索/过滤，进程树只看子进程，拓扑图只是 dagre 层级图。

**方案方向**:

**A. 列表增强**:
- 多列排序（升序/降序）: CPU、内存、PID、名称、端口、启动时间
- 搜索过滤: 按名称/PID/端口/命令/工作目录
- 状态过滤: running/idle/waiting + AI工具/开发服务器/数据库
- 虚拟列表（万级进程不卡顿）

**B. 真实进程勘探**:
- 后端: 递归构建完整进程树（不仅子进程，还有父进程链、兄弟进程）
- 后端: 关联进程的端口绑定、DLL加载、句柄信息
- 前端: 进程详情展开面板（完整信息、环境变量、线程数、句柄数）

**C. 神经关系图**:
- 替换 dagre 静态布局为 d3-force 力导引布局
- 节点间连线为动态脉冲流线（CSS animation + SVG path）
- 粒子效果表示数据流方向和强度（CPU/网络活跃度）
- 节点大小映射资源占用，颜色映射状态
- 鼠标悬停展开关联半径，点击锚定聚焦
- 实时更新：新进程出现有"生长"动画，消失有"衰退"动画

**D. 卡片增强**:
- 信息密度提升: 双列布局、微图表（sparkline）内嵌
- 快捷操作: 一键结束、一键打开目录、一键复制、优先级调整
- 资源趋势: CPU/内存微型趋势图（最近30秒）
- 关联信息: 直接显示绑定端口、关联窗口数

**涉及文件**: `ProcessView.tsx`, `TopologyView.tsx`, `useProcessTopology.ts`, `SystemProcessScanner.ts`, `processHandlers.ts`, 新增 `NeuralGraph.tsx`

---

### 4.3 Spec-03: 端口拓扑深度增强

**问题**: PortRelationshipGraph 372 行，只读无交互，无法聚焦单端口，缺少流量/连接上下文。

**方案方向**:
- 单端口聚焦模式: 点击端口 → 展开关联面板（占用进程、远程连接、历史连接）
- 进程关联: 端口 → PID → 进程详情 → 进程树
- 连接状态可视化: LISTENING/ESTABLISHED/TIME_WAIT 用不同粒子动画
- 网络流向: 本地端口 → 远程地址的流线动画
- 与进程神经图共享力导引引擎（统一的 NeuralGraph 组件）
- 端口组概念: 同一进程的多端口聚合显示

**涉及文件**: `PortRelationshipGraph.tsx`, `PortView.tsx`, `PortScanner.ts`, `portHandlers.ts`, 新增 `PortFocusPanel.tsx`

---

### 4.4 Spec-04: 窗口管理核心增强

**问题**: AI 窗口不可命名，通知不带窗口标识，分组/布局功能异常，进度监控粗糙。

**方案方向**:

**A. AI 窗口自命名系统**:
- 用户可为每个 AI CLI 窗口设定别名（如 "前端重构-Claude", "API开发-Codex"）
- 别名持久化到 electron-store，匹配策略: PID + 窗口标题前缀
- 窗口卡片优先显示别名而非原始标题
- 别名在窗口消失后保留，下次匹配时自动恢复

**B. 通知增强**:
- 任务完成通知格式: `[窗口别名] 任务完成 - {工具名} ({持续时间})`
- 如无别名则显示窗口标题前30字符
- 通知点击直接聚焦到对应窗口
- 通知历史中可快速定位到任务详情

**C. 分组/布局修复**:
- 审计 `WindowView.tsx` 中 groups/layouts tab 的状态管理
- 修复 `createGroup` / `saveLayout` / `restoreLayout` 的 IPC 调用链
- 增加操作反馈（成功/失败 toast）
- 布局预览功能（mini-map 展示窗口位置）

**D. 进度监控增强**:
- AI 任务进度条: 基于多因子评分（CPU、输出模式、时间）的进度估算
- 阶段检测: 启动中 → 思考中 → 编码中 → 验证中 → 完成
- 输出流监控: 通过窗口标题变化检测工作阶段
- 历史趋势: 同类型任务的平均耗时对比

**涉及文件**: `WindowView.tsx`, `AITaskView.tsx`, `WindowManager.ts`, `AITaskTracker.ts`, `NotificationService.ts`, `windowHandlers.ts`, `aiTaskHandlers.ts`, 新增 `AIWindowAlias.tsx`

### 4.5 Spec-05: 主题视觉传达系统

**问题**: 三个主题只是颜色切换，缺少布局/排版/装饰/动效的差异化。

**方案方向**:

**A. 主题维度定义**:

| 维度 | 构成主义 (Constructivism) | 现代光明 (Modern Light) | 温暖光明 (Warm Light) |
|------|---------------------------|------------------------|----------------------|
| **布局** | 不对称、斜切、错位感 | 网格对齐、均衡对称 | 卡片流、圆角柔和 |
| **排版** | 大标题+小正文、强对比 | 均匀字阶、中性层级 | 衬线混排、文学感 |
| **装饰** | 斜线条纹、几何切割、红金色块 | 极简线条、微妙阴影 | 纸质纹理、暖光渐变 |
| **动效** | 机械感平移、齿轮旋转 | 弹性缩放、流畅过渡 | 缓入缓出、自然呼吸 |
| **图标** | 几何化、棱角分明 | 线性、均匀粗细 | 手绘感、圆润 |
| **信息密度** | 高密度、仪表盘风 | 中等密度、留白充分 | 低密度、阅读友好 |
| **数据可视化** | 工业仪表、进度条 | 现代图表、渐变填充 | 有机图形、气泡 |

**B. 实现策略**:
- CSS 层: 主题变量扩展到 layout/spacing/radius/shadow/animation
- 组件层: `ThemeProvider` 提供主题上下文，组件按主题切换渲染变体
- 装饰层: 每个主题独立的 `DecorationSet` 组件（背景图案、角标、分隔线）
- 动效层: 主题相关的 transition/animation presets
- 字体层: 各主题独立字体组合

**C. 组件变体系统**:
```
<Card variant="metric">
  构成主义: 左边红色竖条 + 斜切角 + 工业字体
  现代光明: 圆角卡片 + 微阴影 + 无衬线字体
  温暖光明: 大圆角 + 纸质背景 + 衬线标题
</Card>
```

**涉及文件**: `colors.css`, `globals.css`, `typography.css`, `animations.css`, `useTheme.ts`, 新增 `ThemeProvider.tsx`, `DecorationSet.tsx`, `theme-layouts.css`

## 五、技术架构决策

### 5.1 神经关系图引擎

**选型**: d3-force + React 自定义渲染 (弃用 @xyflow/react 用于动态图)

**理由**:
- ReactFlow (xyflow) 擅长静态/DAG 图，不适合持续动画的力导引图
- d3-force 提供成熟的力模拟引擎（引力、斥力、碰撞、链接力）
- SVG/Canvas 混合渲染: 节点用 SVG（清晰可交互），粒子用 Canvas（高性能）
- 保留 ReactFlow 用于静态拓扑视图作为备选视角

**架构**:
```
NeuralGraphEngine (d3-force simulation)
  ├── NodeRenderer (SVG foreignObject → React components)
  ├── EdgeRenderer (SVG paths with animated dash-array)
  ├── ParticleSystem (Canvas overlay for flow particles)
  ├── InteractionLayer (hover/click/drag handlers)
  └── LayoutController (force parameters, zoom, pan)
```

### 5.2 虚拟滚动

**选型**: @tanstack/react-virtual

**理由**: 轻量、React 原生、支持动态行高

### 5.3 进程数据扩展

后端 `SystemProcessScanner` 扩展:
- 父进程 PID (PPID) 字段
- 兄弟进程列表
- 完整进程链 (→ 祖先到后代)
- 线程数、句柄数
- 关联端口列表（而非单个 port 字段）

### 5.4 主题系统扩展

- CSS Container Queries 支持组件级响应
- CSS `@layer` 分层: tokens → base → components → theme-overrides
- 主题变量扩展: `--layout-*`, `--deco-*`, `--motion-*`

## 七、风险与约束

| 风险 | 影响 | 缓解 |
|------|------|------|
| d3-force + React 性能 | 大量节点时卡顿 | Canvas 粒子 + SVG 节点分层渲染，节点上限 500 |
| PowerShell 调用延迟 | 进程树全关联查询慢 | 增量缓存 + 后台预计算 + 防抖 |
| 主题变体增加维护成本 | 每次 UI 改动需适配 3 主题 | 抽象 `ThemeVariant` 组件层，统一接口 |
| 窗口别名匹配不稳定 | PID 变化后别名丢失 | 多维匹配: PID + 标题前缀 + 命令行 hash |
| Container Queries 兼容性 | Electron 28 是否支持 | Electron 28 = Chromium 120，完全支持 |

## 八、验收标准

### 功能验收

- [ ] 监控面板: 400px~2560px 窗口宽度下数值完整显示无截断
- [ ] 进程列表: 支持至少 6 列的升降序排序
- [ ] 进程搜索: 按名称/PID/端口即时过滤
- [ ] 进程树: 显示选中进程的完整关联图（父→子→孙 + 兄弟 + 端口）
- [ ] 神经关系图: 力导引布局 + 脉冲流线动画 + 交互高亮
- [ ] 端口聚焦: 点击端口展开关联面板
- [ ] AI 窗口: 可设置别名，别名持久化跨重启
- [ ] 通知: 任务完成通知包含窗口别名/名称
- [ ] 分组: 创建/删除/聚焦窗口组正常工作
- [ ] 布局: 保存/恢复/删除窗口布局正常工作
- [ ] 主题: 3 个主题在布局/装饰/动效上有明显差异

### 性能验收

- [ ] 进程列表 500+ 进程滚动流畅 (60fps)
- [ ] 神经图 200 节点动画不卡顿
- [ ] 主题切换 < 300ms 无闪烁
